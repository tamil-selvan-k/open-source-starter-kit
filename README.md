# 🚀 Open Source Starter Kit

Welcome to your very first open-source adventure! 🎉 

This repository is built **specifically for beginners** who want to make their first contribution on GitHub. We know the first step can be intimidating, so we've gamified the process to make it fun, easy, and completely beginner-friendly.

Whether you're fixing a typo, adding your name to a list, or building a small project—you belong here! 

---

## 🎯 The Purpose

Our goal is simple: **Help you make a successful Pull Request (PR) in under 10 minutes.** 
By the end of this guide, you will have successfully contributed to open-source and earned your first contributor badge (metaphorically!).

---

## 🎖️ Contribution Levels

We have structured the contributions into **3 Gamified Levels** so you can pick the one that matches your comfort zone.

### 🟢 Level 1: The First Step (Super Easy)
Just want to get your feet wet? Add your name to our list of contributors!
- **Goal:** Add your name to the `contributors.md` file.
- **Time:** ~2 minutes.

### 🟡 Level 2: The Fixer (A Bit More Involved)
Ready to hunt for small improvements? Help us fix intentional errors!
- **Goal:** Fix a typo, grammar mistake, or formatting issue **ONLY** in the `docs/fix-me.md` file.
- **Important:** All other files in this repository are already correct. Please do not modify them. Only fix content inside `docs/fix-me.md`.
- **How to find issues:** Read through our `docs/fix-me.md` file to spot errors. It's designed to be a mini learning guide!
- **What makes a good fix:** Fixing a spelling mistake like "teh" to "the", improving sentence clarity, or correcting formatting.
- **Rules:**
  1. Fix only ONE issue per Pull Request to give others a chance.
  2. Do not repeat fixes. Check existing Pull Requests first!
  3. Explain your changes clearly in the PR description.

### 🔵 Level 3: The Builder (For the Brave)
Want to show off your skills? Add a mini-project!
- **Goal:** Add a sample project description to the `/projects` folder.
- **How:** Create a new markdown file or use the `sample.md` provided to showcase a cool idea or small app you built. Great for portfolio building!

## 🛠️ How to Contribute (Step-by-Step)

Here is a step-by-step guide on how to contribute directly from the **GitHub Web Interface** (No coding experience needed!)

### 1. Fork the Repository
Click the **Fork** button at the top right of this page to create your own copy of this repository.
![Fork Button](https://via.placeholder.com/800x400?text=Click+the+Fork+Button)

### 2. Go to Your Fork
GitHub will take you to your personal copy of the repository. Make sure the URL shows your username!
![Your Fork](https://via.placeholder.com/800x400?text=Go+to+your+Forked+Repo)

### 3. Create a New Branch
Click the branch dropdown (where it says `main`) and type a new name for your branch (e.g., `add-john-doe` or `fix-typo`). Click "Create branch".
![Create Branch](https://via.placeholder.com/800x400?text=Create+a+New+Branch)

### 4. Edit a File
Navigate to the file you want to edit (e.g., `contributors.md` for Level 1). Click the pencil ✏️ icon in the top right corner of the file view.
![Edit File](https://via.placeholder.com/800x400?text=Click+Pencil+to+Edit)

### 5. Commit Your Changes
After making your edits, scroll down to the "Commit changes" box. Write a short, descriptive message (e.g., "Add Jane Doe to contributors") and click **Commit changes**.
![Commit Changes](https://via.placeholder.com/800x400?text=Commit+Your+Changes)

### 6. Start a Pull Request
Go back to the main page of the original repository (not your fork). You will see a green banner that says "Compare & pull request". Click it!
![Compare & PR](https://via.placeholder.com/800x400?text=Click+Compare+%26+Pull+Request)

### 7. Create the Pull Request
Fill out the provided Pull Request template, explaining what you did, and click the green **Create pull request** button.
![Create PR](https://via.placeholder.com/800x400?text=Create+the+Pull+Request)

🎉 **Congratulations! You just made an open-source contribution!** 🎉

### 💻 Prefer the Command Line? (Git Commands)

If you have Git installed and want to practice terminal commands, here is the quick flow:

```bash
# 1. Fork the repo on GitHub, then clone your fork
git clone https://github.com/<your-username>/open-source-starter-kit.git

# 2. Navigate to the project directory
cd open-source-starter-kit

# 3. Create a new branch
git checkout -b your-branch-name

# 4. Make your changes and stage them
git add .

# 5. Commit your changes
git commit -m "Your descriptive commit message"

# 6. Push to your fork
git push origin your-branch-name
```
Then, go to GitHub and create a Pull Request!
